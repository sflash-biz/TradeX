Add a Trade
Table of Contents
Overview
Use this dialog to add a single trade to your database. The following settings are available when adding trades:

Base Settings
    Return URL - The URL where surfers should be sent
    Status - The status of this trade

    Unconfirmed
    The confirmation e-mail message has been sent, but the trade owner has not yet clicked the link in that message. The trade will not get return traffic or appear in your toplists.

    New
    The trade either needs to be manually enabled by an administrator OR needs to meet the Start requirements set. New trades will not get return traffic or appear in your toplists until they become active.

    Active
    The trade can receive traffic from your site and appear in your toplists.

    Autostopped
    The trade has been stopped by the software because it fell below one of the Minimum requirements. The trade will not get return traffic or appear in your toplists until you enable it.

    Disabled
    You have disabled the trade. The trade will not get return traffic or appear in your toplists until you enable it.
    Color - A custom color to use in the control panel stats displays. This can either be a hex value or color name
    Display in toplist - Include this trade in your toplists
    Grab thumbs - Grab thumbnail images from this trade's site
    E-mail Address - The owner's e-mail address
    Require confirmation - Check this box to require confirmation by e-mail. An e-mail message will be sent to the e-mail address, and the user must click on a link in that e-mail message to confirm their trade account.
    Password - A password for the owner to access their stats. When editing a trade the icon will appear if a password is set for this trade
    Nick - The owner's name/nickname
    ICQ # - The owner's ICQ number
    Site Name - The site's name
    Site Description - A description of the site
    Groups - The group(s) of the site
    Categories - The category(ies) of the site
    Grabber URL - The URL to grab thumbnails from (only if Grab Thumbs is checked)
    Trigger Strings - A comma separated list of strings that should be searched for in the src attribute of <img> tags when grabbing thumbnails.

    These will be used in addition to the global Trigger Strings setting, so it is not necessary to duplicate those values in this field. This field should only be used if you need to setup a custom trigger string for a trade.

    Custom Thumbs - The URLs of thumbnails to use for this trade, one per line
    Banner - The URL of a banner to use for this trade
    Notes - Space to keep notes on this trade, for administrative use only

Forces
    Instant - Use this to force a set number of outs to a trade. Once this number has been reached, the force will be disabled. If you select the high-priority option, this trade will be put on top of the main outlist so it will receive these forced outs sooner. Without that option, the trade will be in the regular forces outlist.
    Hourly - Use this to force a set number of outs to a trade every hour. You can specify the date and time when this force should end, or leave that value blank to have it continue until you manually set this force back to 0.

Trade Settings
    Skim Scheme - Select the skim scheme to use for this trade
    Weight - Assign a weight to this trade. If you want a trade to have a higher priority than others, enter a value larger than 100. To have a lower priority, enter a value less than 100.
    Push To - Use this to set the number of ins you want to push this trade to. This will give the trade an extra out boost so you will get more ins. Once the push value has been reached, the trade will get traffic as normal.
    Push Weight - This weight value will be used for a trade that is being pushed until it reaches it's push to value. Once it does, the regular weight value will take over.
    Start Raws - Specify the number of raw ins a trade must send to become active. This will only affect trades with New status.
    Start Clicks - Specify the number of clicks a trade must generate to become active. This will only affect trades with New status.
    Start Prod - Specify the productivity a trade must reach to become active. This will only affect trades with New status.
    Min Raws - Specify the minimum number of raw ins a trade must send in a 24 hour period to remain active. If the trade drops below this value, it will be autostopped.
    Min Clicks - Specify the minimum number of clicks a trade must generate in a 24 hour period to remain active. If the trade drops below this value, it will be autostopped.
    Min Prod - Specify the minimum productivity a trade must reach to remain active. If the trade drops below this value, it will be autostopped.
    Max Out - Specify the maximum number of outs a trade can receive in a 24 hour period. If this value is exceeded, the trade will not get any more outs until it falls back below this maximum.
    Hourly Cap - Specify the maximum return percentage a trade can reach in a 60 minute period. If this value is exceeded, the trade will not get any more outs until it falls back below this maximum.
    Daily Cap - Specify the maximum return percentage a trade can reach in a 24 hour period. If this value is exceeded, the trade will not get any more outs until it falls back below this maximum.
    Exclude Trades - Select the trade(s) where surfers coming from this site should not be sent.
    External Info - The external info feature is used to make sure that a surfer doesn't get sent back to the same trade even after they have visited many of your trades. For example, if a surfer comes from trade #1, gets sent to trade #2, and then gets sent on to trade #3 from trade #2, the external info feature can tell the script that the surfer has been to all three of these trades so it won't get sent back to them. Each of the trade sites needs to be using a script that supports the External Info feature.
